## 两种多线程Consumer的区别
对应方式  | 优点 |    缺点
  ------------- | ------------- | -------------
 方式一  | 1.容易实现 2.Partition的顺序实现容易 | 如果采用的负载均衡方式是range，那么消费者数量不能超过分区数，否则会出现某些消费者无法消费的状况，其次每个Consumer都需要一个TCP链接，会造成大量的系统性能损耗
 方式二  | 扩展简易，实际上就是一个消费者，拉取到消息后将消息拆分成交给多个线程执行 | 会出现线程安全问题，提交offset等操作尽量放在ConsumerWorker中处理
